# koe-survey 開発ガイド（Claude向け）

## このプロジェクトは何か
「コエ」— クライアント横断で使い回せるエンゲージメント／NPS測定Webアプリ。
設計原則・データ構造・フェーズ計画は **PLAN.md が正本**。変更する前に必ず読むこと。

## 構成
- 素のHTML/CSS/JS（ビルド工程なし・フレームワークなし・CDN依存なし）。この方針は変えない。
- ホスティング＝GitHub Pages（mainのroot）。バックエンド＝Supabase（REST直叩き、`js/api.js`に集約）。
- `js/config.js` が空の間はデモモード（localStorage保存）で動く。デモモードは削除しない（クライアントへのデモと開発時の動作確認に使う）。

## 変えてはいけないもの（ガードレール）
1. **匿名性の設計**：回答にIP・氏名・時刻を持たない（日付のみ）／n<5の属性クロス非表示。緩める変更は提案のみにして実装しない。
2. **supabase/schema.sql のRLS**：anonはINSERTのみ・列挙不可（RPC経由）。変更が必要なら理由を明記してPRの説明に書く。
3. **service_roleキーやパスワードをリポジトリに置かない**。anon keyはOK。
4. **クライアントの実名を入れない**：このリポジトリはpublic。サーベイのslug・client欄・コミットメッセージも仮名（例：nd）を使う。実データ・実名の設問調整はSupabase側で行う。

## 開発の作法
- UI文言・コメントは日本語。トーンは丁寧・簡潔（クライアントに見せられる品質）。
- 動作確認：`python3 -m http.server 8000` → `#/s/demo-nps`（回答画面）と `#/admin`（管理画面・「デモ回答を30件生成」ボタンで集計確認）。
- JS構文チェック：`node --check js/*.js`／JSON検証も忘れずに。
- チャートは `js/charts.js` の自前SVG。配色（青#2a78d6・NPS極性の青/灰/赤）は検証済みパレットなので変えない。
- コミットは小さく・日本語メッセージで。mainへ直接pushせずPRを作る。
